Overall Verdict: SAFE TO USE

RTK is a 100% offline CLI tool. It contains zero network dependencies, makes zero external connections, sends no telemetry or analytics, and stores all data locally on your machine. The codebase is transparent, open-source (MIT license), and has been audited across 57 Rust source files (~50,249 lines of code).

Key risks are limited to information loss from aggressive filtering (output truncation/summarization) and minor local data persistence (command history in SQLite). Both are configurable and can be disabled.

What RTK Does (Plain English)

  1. Intercepts CLI commands — When you run rtk git log, it runs git log under the hood
  2. Captures the output — stdout and stderr are piped through RTK
  3. Filters/compresses the output — Removes noise (ANSI codes, verbose logs, passing tests, etc.)
  4. Returns condensed output — 60-90% fewer tokens reach your LLM context
  5. Tracks savings locally — Records how many tokens were saved in a local SQLite database

RTK never modifies your code, deletes files, or changes command behavior. It only transforms the output before it reaches the LLM. The underlying command always executes with full fidelity.
